# 포트폴리오용 캡처: 직접 찍어야 하는 화면 체크리스트

아래는 제가 대신 만들 수 없는, 실제 앱을 켜서 직접 캡처해야 하는 화면들입니다
(이미지 생성이 아니라 실제 동작 증거이기 때문에 직접 찍는 게 맞습니다).

## 1. 추천 결과 화면 (필수, 우선순위 1위)
- 웹 데모(`web/static/index.html`) 또는 실기기 Expo Go 화면
- 재료 수량(`amount`/`perServingAmount`)이 보이는 상태로 — 이미 보내주신 닭가슴살
  파슬리 구이 스크린샷이 이 조건을 충족합니다, 그대로 써도 됩니다
- 가능하면 "⏰ 임박재료 사용" 배지가 뜬 레시피가 하나 포함된 결과면 더 좋음(검증
  로직이 실제로 동작하는 증거)

## 2. 검증 실패 → 재시도 → 통과 배너
- `validate_banner.fail`(⚠️ 검증 미통과 · 재시도 N회) 상태가 뜬 화면을 한 번 캡처해두면,
  "검증 로직이 실제로 무언가를 잡아낸다"는 증거가 됩니다(항상 통과만 하면 검증이 의미
  없어 보일 수 있음). 일부러 까다로운 조합(재료 1~2개만, microwave 환경)으로 추천을
  걸면 재현될 가능성이 높습니다

## 3. 모바일 실기기 화면 (iPhone, Expo Go)
- 온보딩(가구원 수 슬라이더) 1장
- 냉장고 탭(재료 추가 + "곧 먹어야 해요" 토글 켠 상태) 1장
- 추천 탭 결과 1장
- 세 장을 나란히 배치하면 "모바일 앱으로 끝까지 연결됐다"는 차별점이 한눈에 보입니다
